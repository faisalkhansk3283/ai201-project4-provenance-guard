# Provenance Guard — Planning Document

**Project:** AI content attribution system for creative writing platforms  
**Stack:** Flask, Groq (llama-3.3-70b-versatile), SQLite, Flask-Limiter  
**Status:** Planning complete — ready for implementation

---

## Architecture

### Submission Flow

```
POST /submit
  { text, creator_id }
        │
        ▼
  ┌─────────────────────────────────────┐
  │         Detection Pipeline          │
  │                                     │
  │  Signal 1: LLM Judgment (Groq)      │
  │  Signal 2: Stylometric Analysis     │
  │  Signal 3: Informality Scoring      │
  └─────────────────────────────────────┘
        │
        ▼
  Confidence Scoring
  (weighted average of 3 signals → 0.0 to 1.0)
        │
        ▼
  Transparency Label Generation
  (based on confidence thresholds)
        │
        ▼
  Audit Log (SQLite)
        │
        ▼
  Response
  { content_id, attribution, confidence, label, signals }
```

### Appeal Flow

```
POST /appeal
  { content_id, creator_reasoning }
        │
        ▼
  Lookup content_id in database
        │
        ▼
  Update status → "under_review"
        │
        ▼
  Append appeal to audit log
        │
        ▼
  Response { message, status, content_id }
```

### Architecture Narrative

A submitted piece of text passes through three independent detection signals — one semantic (LLM), one structural (stylometric math), one linguistic (informality markers). Their scores are combined into a single confidence score (0.0 = human, 1.0 = AI). That score maps to a transparency label shown to readers. Every decision is written to a structured SQLite audit log. If a creator disputes the classification, they submit an appeal with their reasoning — the system updates the status to "under review" and logs the appeal without automatic re-classification.

## Detection Signals

### Signal 1: LLM Judgment (Groq)

**What it measures:** Semantic coherence, stylistic uniformity, and overall writing pattern. The LLM assesses holistically whether the text reads as human or AI-generated — capturing things like unnatural smoothness, lack of personal voice, and overly balanced sentence structure.

**Output:** Float between 0.0 (human) and 1.0 (AI)

**How:** Send text to Groq with a structured prompt asking for a JSON response with `ai_probability` field.

**Blind spots:** Formal human writing (academic papers, legal text) may score high. Casual AI output with deliberate errors may score low. Short texts give the LLM less signal to work with.

---

### Signal 2: Stylometric Analysis

**What it measures:** Statistical properties of writing that differ between humans and AI. Specifically:
- **Sentence length variance** — humans vary sentence length dramatically; AI is uniform
- **Type-token ratio (TTR)** — vocabulary diversity; AI reuses words more predictably
- **Average sentence length** — AI tends toward medium-length sentences consistently

**Output:** Float between 0.0 (human-like variation) and 1.0 (AI-like uniformity)

**How:** Pure Python math — no external libraries. Split text into sentences and words, compute variance and ratios.

**Blind spots:** Academic or technical human writing has naturally low variance. Very short texts (under 3 sentences) produce unreliable variance scores.

---

### Signal 3: Informality Score (Ensemble)

**What it measures:** Presence of casual human language markers that AI almost never produces naturally:
- Informal shorthand: `ok`, `gonna`, `wanna`, `kinda`, `gn8`, `tc`, `brb`, `lol`
- Informal contractions: `i'm`, `don't`, `can't`, `it's` (especially lowercased)
- Casual punctuation: `...`, `!!`, `??`
- Sentence starters: `And`, `But`, `So`, `Cause`
- Typos or intentional misspellings

**Output:** Float between 0.0 (very informal = human) and 1.0 (very formal = AI-like)

**How:** Keyword matching + punctuation pattern counting in pure Python.

**Blind spots:** Formal human writing scores as AI-like. A human writing a professional cover letter will appear AI-generated by this signal alone.

## Confidence Scoring

### Combining Signals

We use a **weighted average** of the three signals:

| Signal | Weight | Reasoning |
|--------|--------|-----------|
| LLM Judgment | 55% | Most holistic, captures semantic patterns |
| Stylometric | 35% | Structural, independent of LLM |
| Informality | 10% | Useful signal but too narrow for formal writers |

```
confidence = (0.55 × llm_score) + (0.35 × stylo_score) + (0.10 × informality_score)
```

### Thresholds

We bias toward protecting human writers — a false positive (calling human work AI) is worse than a false negative.

| Confidence Score | Attribution | Label Variant |
|-----------------|-------------|---------------|
| 0.0 – 0.35 | likely_human | Likely human written |
| 0.36 – 0.74 | uncertain | Uncertain |
| 0.75 – 1.0 | likely_ai | Likely AI generated |

### What 0.6 means

A score of 0.6 means the system sees meaningful AI-like patterns but is not confident enough to label it as AI-generated. The two signals are likely disagreeing — perhaps the LLM scores it high but the informality signal scores it low. The user sees the "Uncertain" label, and the creator retains full ability to appeal.

## Transparency Labels

Three label variants — exact text shown to readers:

### High-confidence AI (score ≥ 0.75)
```
⚠️ AI Generated
This content shows strong patterns of AI generation (confidence: {score}%).
This is automated analysis — the creator may appeal this classification.
```

### Uncertain (score 0.36 – 0.74)
```
❓ Uncertain
Our system could not confidently determine authorship (confidence: {score}%).
The creator may appeal this classification.
```

### High-confidence Human (score ≤ 0.35)
```
✅ Human Written
This content shows strong patterns of human authorship (confidence: {score}% human).
```

## Appeals Workflow

**Who can appeal:** Any creator who submitted the content (identified by `creator_id`).

**What they provide:**
- `content_id` — the ID of the submission they are appealing
- `creator_reasoning` — their explanation in plain text

**What the system does:**
1. Looks up the `content_id` in the database
2. Updates status from `"classified"` to `"under_review"`
3. Appends appeal record to audit log with reasoning and timestamp
4. Returns confirmation to the creator

**What a human reviewer sees in the appeal queue:**
- Original submission text
- All three signal scores
- Combined confidence score
- Creator's reasoning
- Timestamp of original classification and appeal

**Automated re-classification:** Not implemented. A human reviewer makes the final call.

## Edge Cases

### Edge Case 1: Formal human writing
A professor submitting an academic essay will have low sentence variance, formal vocabulary, and no informal markers — all three signals will push toward AI. Our system will likely misclassify this. The higher threshold (0.75) reduces this risk but does not eliminate it.

### Edge Case 2: Deliberately casual AI output
If someone prompts an AI to "write like a teenager texting," the output will contain informal markers and intentional errors — the informality signal will score it as human even though it's AI-generated. Our system cannot reliably detect this.

### Edge Case 3: Very short text
Texts under 3 sentences give Signal 2 (stylometric) almost no data — variance of 1-2 sentences is meaningless. Short texts will rely heavily on Signal 1 (LLM) and Signal 3 (informality).

### Edge Case 4: Non-native English speakers
Non-native speakers often write in patterns that resemble AI — consistent grammar, limited vocabulary range, formal phrasing. Our system may flag their work unfairly. The appeals workflow is the primary mitigation.

## API Endpoints

### POST /submit
**Request:**
```json
{
  "text": "string — the content to analyze",
  "creator_id": "string — unique identifier for the creator",
  "content_type": "text | image_description (optional, defaults to text)"
}
```
**Response:**
```json
{
  "content_id": "uuid",
  "attribution": "likely_ai | uncertain | likely_human",
  "confidence": 0.82,
  "label": "⚠️ Likely AI-Generated ...",
  "signals": {
    "llm_score": 0.85,
    "stylometric_score": 0.76,
    "informality_score": 0.71
  },
  "status": "classified"
}
```

**Request:**
```json
{
  "content_id": "uuid",
  "creator_reasoning": "string"
}
```
**Response:**
```json
{
  "message": "Appeal received. Your submission is now under review.",
  "content_id": "uuid",
  "status": "under_review"
}
```

## Stretch Features

### 1. Ensemble Detection (3rd Signal)
**Plan:** Implement Signal 3 (Informality Score) as documented above. Weight it at 20% in the combined score. Document the three-signal weighting approach in README.

### 2. Provenance Certificate
**Plan:** Add a `POST /verify` endpoint where a creator can submit additional proof of authorship (a written statement + their creator_id). If accepted, their submissions get a `"verified_human"` badge in the response and audit log. Store verified creator IDs in the database.

### 3. Analytics Dashboard
**Plan:** Add a `GET /analytics` endpoint returning:
- Total submissions
- Attribution distribution (how many likely_ai / uncertain / likely_human)
- Appeal rate (appeals / total submissions)
- Average confidence score
- Most common attribution by hour (trend)

### 4. Multi-modal Support
**Plan:** The `/submit` endpoint already accepts `content_type` field. When `content_type = "image_description"`, apply adjusted signal weights — informality signal weight drops (captions are naturally formal) and LLM weight increases. Signal 2 (stylometric) uses character count thresholds adjusted for short captions.
